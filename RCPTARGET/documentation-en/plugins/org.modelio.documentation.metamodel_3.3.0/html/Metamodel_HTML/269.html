GeneralOrdering
(not implemented) Describes that one OccurrenceSpecification must occur before the other.
NOT IMPLEMENTED IN SEQUENCE DIAGRAMS. This mechanism provides the ability to define partial orders of OccurrenceSpecifications that may otherwise not have a specified order. A GeneralOrdering may appear anywhere in an Interaction, but only between OccurrenceSpecifications.
