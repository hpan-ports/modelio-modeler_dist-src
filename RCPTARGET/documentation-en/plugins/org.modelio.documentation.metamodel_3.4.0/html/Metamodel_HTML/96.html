resources
Resources occurences are attached to the most important model elements, and conform to generiquely defined resources. They can correspond to indicators, resource consumption, ownership, etc. Resources are defined elsewhere, and can correspond to specific modeling approches, simulation techniques, etc. This model has been generalized compared to the BPMN2 model, more focused on activity resources. A FAIRE : Ce modèle doit être retouché. Où sont définies les ressources génériques? (ownership), formaliser davantage les types : booléen, entier, énum, texte, ... Ce modèle correspond à peu près au principe des propriétés du module Analyst. Une réflexion doit être engagée : les fusionne t'on avec analyst? énéralise t'on ceci à tous les model element?
